## Motivação
- Minha opinião: Aprender lógica, como funções funcionam, condicionais, laços de repetição, busca binária e Big O (analise de complexidade) já um grande começo para ser "poliglota de linguagens de programação", tendo esses conceitos bem consolidados ao aprender uma nova linguagem seria necessário aprender apenas as suas particularidades e o que não se pode fazer naquela linguagem
	- Ex de particularidade: 
		- Em cada linguagem existe uma forma de imprimir uma informação na tela:
			- Em Java:
			```Java
			System.out.println()
			```
			- Em C:
			```C
			printf()
			```
			- Em C++:
			```C++
			cout()
			```
			- Em Python:
			```Python
			print()
			```
			- Em JavaScript:
			```JavaScript
			console.log()
			```
			- Cada uma possui uma sintaxe diferente (um modo de escrever a mesma coisa) porém quando o conceito de  "imprimir na tela" (output) é entendido se sabe quando utiliza-lo, aprender uma nova linguagem muitas vezes significa apenas aprender a nova sintaxe para fazer a mesma coisa.
	- O ';' também é uma particularidade de algumas linguagens, em C, C++ e Java é obrigatório colocar o ; no fim de uma instrução (esquecer isso é um crime, gera erro), já em JavaScript o ';' é opcional e em Python ele não é usado 
	- Cada uma das linguagens citadas acima tem também seus casos de uso (onde seria mais inteligente usa-las):
		- Python: Ciências de dados e IA
		- C: Em sistemas embarcados e softwares de baixo nível
		- C++: Jogos
		- Java: Aplicações android e backend
		- JavaScript: Programação web
- Ou seja, saber paradigmas e conceitos permite compreender melhor como cada linguagem funciona e em quais situações utilizá-la por isso programadores experientes geralmente conseguem aprender novas linguagens com mais facilidade: eles já dominam os conceitos, precisando aprender principalmente a sintaxe e as particularidades da nova linguagem

## Critérios de avaliação de linguagens de programação

- Os 4 principais critérios de avaliação de linguagens de programação:
	1. Legibilidade: Facilidade de ler e entender o código
	2. Facilidade de escrita: Quão fácil é programar na linguagem
	3. Confiabilidade: O quanto podemos confiar que um programa escrito naquela linguagem vai funcionar corretamente e produzir resultados corretos
	4. Custo: Custo total (desde o custo para aprender até o custo de erros)
##### Legibilidade
- Antes as linguagens de programação eram mais legíveis para as máquinas do que para os humanos (Assembly)
- A legibilidade é um fator muito importante para a manutenção de softwares 
- Código difícil de ler = difícil de corrigir e evoluir
- **Características que contribuem para a legibilidade:** 
	- **Simplicidade geral:** Para que a simplicidade seja boa é necessário que haja poucas regras e poucas formas diferentes de fazer a mesma coisa. 
		- Exemplo de simplicidade ruim:
		```Java
		count = count + 1;
		count += 1;
		count++;
		++count;
		```
		- Varias maneiras diferentes de fazer a mesma coisa, de adicionar 1 a uma variável contadora. Um programador pode não entender que ```++count```significa o mesmo que ```count++``` ou algum dos outros modos
	
	- **Ortogonalidade:** Se algo funciona em um lugar, normalmente funciona em qualquer outro lugar.  Uma linguagem ortogonal permite que combinar estruturas de dados
		- Ortogonalidade é mais fácil de entender com exemplos:
		- Exemplo de linguagem ortogonal: Em Python qualquer coisa pode ser guardada em uma variável (e sempre funciona)
		```Python
		x = 10
		x = "oi"
		x = [1,2,3]
		```
		- Além disso em Python funções recebem qualquer tipo de dado
		```Python
		def mostrar(x):
		    print(x)
		    
		mostrar(10)
		mostrar("oi")
		mostrar([1,2,3])
		```
		- Exemplo de linguagem não ortogonal: C, em C é preciso definir o tipo da variável, isso gera mais exceções e mais regras (para cada tipo de variável)
		```C
		int x = 10;
		char d[20]; //string
		```
		- Em C para usar a função demonstrada anteriormente (mostrar(x)) é necessário criar várias versões (uma para cada tipo de dado)
		```C
		void mostrarInt(int x)
		void mostrarString(char x[])
		```
		- Outra questão de falta de ortogonalidade em C é que funções não podem retornar arrays
	
	- **Tipos de dados:**  Escolher o tipo de dado adequado para a estrutura
		- Exemplo: Você quer que o usuário entre com uma letra ('y' ou 'n') não faz sentido você criar uma String e sim uma variável do tipo char (assim você economiza memória, alguns bytes), outro caso também é quando se tem um número decimal com poucas casas decimais (ex: 3.14) o correto é declarar ele como um float e não como double (double são números decimais com mais casas decimais e float com menos). Isso não é vantajoso apenas para memória mas para a legibilidade também, quando alguém lê uma variável char sendo declarada ele já associa que é apenas uma letra que será armazenada e ao ver um float sabe que se trata de um número decimal com menos casas decimais ou menor precisão

	- **Projeto da Sintaxe:** Sintaxe é a forma de escrever cada coisa, cada linguagem tem uma maneira diferente de escrever algo, algumas linguagens possuem estruturas mais claras

##### Facilidade de Escrita
- Quão fácil é programar soluções na linguagem
- **Características que contribuem para a facilidade de escrita:** 
	- **Simplicidade:** Menos regras programação mais fácil
	
	- **Ortogonalidade**: Menos exceções menos coisas para memorizar
	
	- **Abstração:** Colocar detalhes complexo "debaixo dos panos". Linguagens orientadas a objeto são muito abstratas (abstração de dados) (criasse um objeto (classe)). Também existe a abstração de processos que é chamar uma função (sem precisar repetir código)

	- **Expressividade:** Quando uma linguagem permite fazer muito com pouco código

##### Confiabilidade 
- Um programa executa corretamente em todas as condições previstas, detecta erros durante a compilação ou execução
- **Fatores que aumentam confiabilidade:**
	- **Verificação de tipos:** A linguagem verifica se os tipos são compatíveis. Em C uma variável de tipo int não pode receber um valor do tipo String. Já em JavaScript é possível fazer a seguinte coisa: total = 100 + "10" que vai gerar o resultado 10010 em vez de 110 (está adicionando um int com uma string aí "une" os dois em vez de somar)
	
	- **Tratamento de erros:** São os try/catch, que captura um erro em tempo de execução. Onde o Try pega o trecho que pode dar o erro (Try é meio que "tente fazer isso") já o catch ocorre após o try resultar em erro (ele trata esse erro), Sem tratamento de erro o programa poderia travar. Exemplo:
	```Python
	try:
    x = 10 / 0
	except:
    print("Erro de divisão")
	```

	- **Utilização de Apelidos (Aliasing):** Variáveis diferentes, mas apontam para o mesmo dado então, alterar uma variável altera a outra. Exemplo: (ponteiros em C)
	```C
#include <stdio.h>

int main()
{
	int a = 5;
	int *b = &a;
    
    *b = 20;
	printf("%d", a);
}

//ao imprimir a variavel 'a' o resultado será 20
	```

	3. Custo de compilar programas na linguagem
	4. Custo de executar programas
	5. Custo do sistema de implementação da linguagem
	6. Custo de uma confiabilidade baixa
